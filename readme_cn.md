```
\#\# 项目背景  
  
传统钻石证书（如 GIA、IGI）存在以下问题：  
  
- \*\*易伪造\*\*：纸质或 PDF 证书容易被篡改或复制。  
- \*\*标准不一\*\*：不同实验室对颜色、净度的判定存在主观差异。  
- \*\*流转不透明\*\*：钻石从矿场到零售商的链条中，证书与实物可能脱节。  
- \*\*资产孤岛\*\*：证书只是一张静态凭证，无法参与数字金融生态。  
  
BGI 通过以下方式解决：  
  
- 将钻石的关键属性（4C + 荧光 + 区块链指纹）写入 NFT。  
- 证书元数据存储在去中心化网络中，确保永久可验证。  
- 通过智能合约实现证书的发行、转让、注销的全生命周期管理。  
- 将钻石证书升级为可编程、可流通的数字资产。

传统 4C 标准仅描述钻石的物理属性（重量、净度、颜色、切工）。BGI 的 \*\*6C 标准\*\* 在此基础上增加 \*\*荧光（Fluorescence）\*\* 与 \*\*区块链指纹（Chain Fingerprint）\*\*，实现从物理真实性到数字可信度的完整价值闭环。  
  
| 维度 | 传统 4C 标准 | BGI 6C 标准 | 核心价值升级 |  
| :--- | :--- | :--- | :--- |  
| \*\*评估体系\*\* | 重量、净度、颜色、切工 | 4C + 荧光 + 区块链指纹 | 全面描述钻石的物理特性与数字身份 |  
| \*\*信任机制\*\* | 依赖鉴定机构信誉 | 依赖区块链不可篡改的数学算法 | 从“机构背书”升级为“技术信任” |  
| \*\*溯源能力\*\* | 静态报告，信息孤立 | 全生命周期链上追踪 | 实现从矿场到指尖的端到端透明溯源 |  
| \*\*真伪验证\*\* | 证书与钻石分离，存套用风险 | 物理特征哈希上链，强绑定 | 从“人眼对比”升级为“算法验证” |  
| \*\*资产属性\*\* | 一次性购买凭证 | 可编程、可流通的数字资产 | 打开 DeFi 借贷、资产代币化等金融场景 |

\#\#\# 1. 钻石 6C 数据的数字化映射  
  
BGI 将 6C 数据采用标准化 JSON Schema 描述，并附上分级师签名：  
  
\`\`\`json  
\{  
  "version": "1.0",  
  "diamond\_id": "BGI-210394",  
  "carat\_weight": 1.24,  
  "color": "D",  
  "clarity": "VVS1",  
  "cut": "Excellent",  
  "fluorescence": "None",  
  "chain\_fingerprint": "0x7a3f...c9e2",  
  "grading\_lab": "BGI Hong Kong",  
  "grading\_date": "2026-04-01",  
  "grader\_signature": "0x9d2e..."  
\}
```

- **`chain\_fingerprint`**：由钻石腰棱激光微刻码 + 高倍显微镜图像的哈希值生成，确保实物与证书唯一绑定。

- **分级师签名**：使用 HSM 签名，确保数据来源真实。

### 2. NFT 证书铸造流程

1. **实物分级**：BGI 授权分级师使用标准设备测量钻石，生成 6C 数据。

2. **数据签名**：分级师使用其私钥对数据包进行签名，确保来源真实性。

3. **上传 IPFS**：签名后的数据包上传至 IPFS，获得唯一 CID（`Qm...`）。

4. **铸造 NFT**：调用智能合约的 `mintCertificate` 方法，传入：

   - 钻石 CID

   - 分级师签名

   - 所有者钱包地址

5. **链上记录**：智能合约生成 ERC-721 Token，将 `tokenId` 与 CID 绑定，并记录铸造时间、分级师地址。

### 3. 去中心化存储与内容寻址

- **原始数据**（分级图像、光谱数据）存储于 IPFS。

- **证书元数据**（符合 ERC-721 Metadata 扩展）指向 IPFS CID。

- **持久化保障**：通过 Filecoin 存储交易或 Pinata 固定服务，确保证书数据不丢失。

- **完整性验证**：任何下载的证书文件均可通过 `ipfs cid compute` 重新计算哈希，若与链上记录的 CID 一致，则证明证书未被篡改。

### 4. 智能合约验证机制

BGI 智能合约提供以下核心接口：

| 函数名 | 作用 |
| - | - |
| `verifyCertificate(tokenId)` | 返回证书 CID、分级师地址、铸造时间、当前所有者 |
| `isCertificateValid(tokenId)` | 检查证书是否被注销或转让（仅记录最新所有者） |
| `transferCertificate(to, tokenId)` | 转让证书所有权，自动更新链上记录（需原所有者签名） |
| `revokeCertificate(tokenId)` | 仅合约管理员（BGI 多签钱包）可调用，用于作废伪造证书 |


**链上校验逻辑示例**（Solidity）：

```
function verifyCertificate(uint256 tokenId)  
    public view returns (bool, string memory, address)  
\{  
    Certificate storage cert = certificates\[tokenId\];  
    require(cert.exists, "Token does not exist");  
    require(!cert.revoked, "Certificate revoked");  
    return (true, cert.cid, cert.originalGrader);  
\}
```

## 安全与隐私设计

- **分级师身份锚定**：分级师的公钥与 BGI 颁发的链上 DID（去中心化身份）绑定，每份证书签名均可验证。

- **钻石指纹防偷换**：`chain\_fingerprint` 由钻石腰棱激光微刻码 + 高倍显微镜图像的哈希组成，确保证书与实物强绑定。

- **隐私保护**：证书不包含所有者姓名等个人信息，仅记录钱包地址。零售环节可通过零知识证明（ZKP）选择性披露部分属性（如“净度 ≥ VS2”）。

- **多签管理**：核心合约升级、证书批量注销等操作需要 3/5 多签钱包批准，防止单点风险。


## 使用指南

### 查验钻石证书（无需钱包）

1. 访问 [bgi.xyz/verify](https://bgi.xyz/verify)

2. 输入钻石证书上的 `tokenId` 或扫描二维码

3. 页面将显示：

   - 6C 完整数据

   - 分级师签名状态（有效/无效）

   - 当前所有者钱包地址（匿名）

   - 证书铸造时间及区块链确认数

### 铸造新证书（仅授权分级师）

- 分级师登录内部系统 → 连接硬件钱包 → 上传分级数据 → 自动生成签名并调用智能合约。

### 转让证书（钻石所有者）

- 在 [bgi.xyz/transfer](https://bgi.xyz/transfer) 连接钱包 → 输入接收方地址和 `tokenId` → 确认交易（支付少量 Gas 费）。
